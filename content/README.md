# content

The recurring content stream. The `content-pipeline` playbook writes here, **one dated subfolder per run** (`content/<YYYY-MM-DD>/`), with one draft per channel defined in `memory.md`.

This is a named home, not a catch-all — content has a fixed, predictable destination so scheduled runs always land in the same place. Project-specific deliverables don't go here; they live inside their project folder in `01-projects/`.
