# Playbook: remember

**Use when:** I correct you, state a standing rule, or tell you a fact worth keeping — "remember this", "don't do that again", "from now on…". Captures it so it never has to be re-explained.

**Inputs:** the fact, correction, or rule (usually said in passing).

## Steps

1. Classify it: identity/account, voice/style, standing decision, automation, gotcha, or project/person.
2. Write it as one short declarative line.
3. Append it to the matching section of `memory.md`. If it contradicts an existing line, replace that line rather than adding a duplicate.
4. If the correction is really about *how a playbook should run*, also update that playbook's steps — and say so.
5. Confirm what you saved and where.

## Output

A new (or replaced) line in `memory.md`, and optionally an updated playbook. The fact is now persistent — read at the start of every future session.
