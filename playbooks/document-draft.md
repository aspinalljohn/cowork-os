# Playbook: document-draft

**Use when:** you need a first draft of a document — memo, post, report, proposal — from a brief or source material.

**Inputs:** a one-line description of what to write, plus any source files in `00-inbox/`, `01-projects/`, or `02-reference/`.

## Steps

1. Confirm the essentials before drafting: **audience**, **purpose**, **length/format**, **what it must include**. Ask if any are missing — don't guess on these four.
2. Pull supporting material from the named sources. Cite anything factual.
3. Draft in my voice (see voice profile in `../SKILL.md`). Lead with the point; structure with clear headers.
4. Mark anything you're unsure of with `[CHECK: ...]` rather than asserting it confidently.
5. Write the draft **inside the project it serves**: `01-projects/<project>/<slug>-<YYYY-MM-DD>.md` (create a project-of-one folder if it's standalone). Tell me what you'd tighten on a second pass.

## Output

A complete first draft inside the relevant `01-projects/<project>/` folder, in my voice, with `[CHECK]` flags on anything uncertain and a short note on what to revise next.
