# Playbook: recall

**Use when:** before building anything new, I want to know what the workspace already holds on a topic — so I reuse instead of re-deriving. The Obsidian-free version of searching the vault: you read the files instead of querying a graph.

**Inputs:** a topic or question.

## Steps

1. Search across `memory.md`, `02-reference/`, `01-projects/`, `content/`, `reviews/`, and `playbooks/` for anything related to the topic.
2. Read the matches enough to summarize what they contain.
3. Report three things: **what I already know** (existing notes/outputs), **what assets exist** (playbooks, reference, prior drafts I can reuse), and **the gap** (what isn't covered yet).
4. Recommend the fastest path: reuse X, extend Y, or build new.

## Output

A short briefing in chat — known / assets / gap / recommendation — so I start from what exists instead of a blank page. Nothing written.
