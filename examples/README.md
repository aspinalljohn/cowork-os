# Worked example — the Sam Rivera workspace

This folder is a **filled-in example** so you can see Cowork OS already running before you set up your own. It belongs to a fictional persona — **Sam Rivera, an independent go-to-market consultant** — and shows what the files look like once they have real content in them.

**Copy the shape, not the facts.** Your own `memory.md` and `map.md` live one folder up (in the workspace root); these are reference only. Nothing in here is wired into your live workspace.

## What to look at

| File | Shows you |
|---|---|
| `memory.md` | A real, filled-in memory — identity, voice rules, content channels, automations. This is the single most useful file to model yours on. |
| `map.md` | What `workspace-sweep` produces once the workspace has content (the dataview replacement, populated). |
| `reviews/daily-review-2026-06-19.md` | What the `daily-review` playbook leaves you each morning. |
| `01-projects/fintech-launch/research-brief-saas-pricing-models-2026-06-18.md` | What `research-brief` produces — filed *inside the project it informs*, not a separate outputs bin. |
| `content/2026-06-19/` | What the flagship `content-pipeline` produces — one draft per channel, in Sam's voice. |

## How to use it

1. Read `examples/memory.md` next to your blank `memory.md` — fill yours in the same shape.
2. Skim the sample outputs to calibrate what "done" looks like for each playbook.
3. Delete this `examples/` folder before you share or go live, if you want a clean workspace — or keep it as an onboarding reference for the next person.
